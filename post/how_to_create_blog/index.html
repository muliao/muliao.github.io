按照自己的搭建过程描述

以下操作都是在win10的ubuntu子系统上进行

创建出github的blog仓库

create repository

仓库名为 muliao.github.io

创建后访问https://muliao.github.io就可以看到一个空页面了

使用hugo作为博客页面生成工具

在我们开始之前，我们应该检查我们的Ubuntu机器的架构，以便我们可以确保下载正确的软件包。 在服务器上，键入：

$ uname -i
x86_64

说明服务器是ubuntu 64位,

进入 https://github.com/gohugoio/hugo/releases 寻找hugo的64位最新版本

这时找到的最新版本的地址是https://github.com/gohugoio/hugo/releases/download/v0.49/hugo_0.49_Linux-64bit.deb

下载:

$ wget ttps://github.com/gohugoio/hugo/releases/download/v0.49/hugo_0.49_Linux-64bit.deb
$ sudo dpkg -i hugo_0.49_Linux-64bit.deb
$ hugo version
Hugo Static Site Generator v0.49 linux/amd64 BuildDate: 2018-09-24T10:03:17Z

指令输出的版本正确的话，说明安装完成了

使用Jane主题

jane主题地址 https://github.com/xianmin/hugo-theme-jane

按照上面的介绍搭建一个基本的blog框架

$ hugo new site muliao_blog
$ cd muliao_blog
$ git clone https://github.com/xianmin/hugo-theme-jane.git --depth=1 themes/jane
$ cp -r themes/jane/exampleSite/content ./
$ cp themes/jane/exampleSite/config.toml ./
$ hugo server

此时就启动了一个带有例子的blog了,通过访问 http://localhost:1313 可以看到一些博客用例了。

自己搭建的时候发现用例上的代码高亮和jane官网上的不一样效果，折腾了很久都没发现哪里有问题。

通过其他途径解决这个难题，还好发现了以下这个地方 http://note.qidong.name/2017/06/24/hugo-highlight/

修改themes/jane/layouts/partials/custom_head.html,增加以下内容

<link rel="stylesheet" href="https://cdnjs.cloudflare.com/ajax/libs/highlight.js/9.12.0/styles/github.min.css">
<script src="https://cdnjs.cloudflare.com/ajax/libs/highlight.js/9.12.0/highlight.min.js"></script>
<script>hljs.initHighlightingOnLoad();</script>

比较好的解决了代码格式/高亮的问题，可以通过 http://highlightjs.org/ 上面的介绍来做选择不同的风格

添加评论

Hugo没有内置评论功能，要增加评论功能需要集成第三方评论服务，比如国外最流行的Disqus。hugo主题内置了disqus评论插件，不过需要你按如下操作配置一下，否则页面下方的disqus插件总是显示无法连接。

获取disqusShortname

1.注册disqus

官网注册帐号 https://disqus.com/

这里用disqus主账号不行，需要用主账号login后：add a newsite to disqus，比如加入muliao.disqus.com，这样你的disqusShortname就为muliao

创建选择installation平台的配置时选择universal code,然后 将生成的code放到覆盖到themes/jane/layouts/partials/comments.html中

在config.toml中配置disqusShortname:

disqusShortname = "muliao"

此时打开刚刚的用例博客，就可以看到底部可以评论了

可以对某篇博文单独关闭disqus功能。如test.md

---
Categories = ["test"]
Description = "test"
Tags = ["test"]
title = "test hugo"
comments = false      #这里的comments取值为false，disqus则不可见
---

当然在 https://disqus.com 也可以管理这些评论

可以创作自己的博客了

这时可以删除掉content/post下面的所有的用例博客了，要建立自己的内容了。

新建一个how_to_create_a_blog.md文件,写入本篇文章(当然是用md格式)

修改config.toml为自己的配置

$ hugo server

访问本地服务就可以看到自己的这篇文章了，觉得没问题，就可以发布到github上了。

发布博客到自己的github上

在最前面已经创建好了仓库了,现在只要克隆到muliao_blug/public目录下

$ cd muliao_blog
$ git clone https://github.com/muliao/muliao.github.io.git public
$ hugo
$ cd public
$ git add .
$ git commit -m '发表了第一篇文章'
$ git push

hugo指令生成静态页面到public，此时上传这些文件即可

访问 https://muliao.github.io 是不是可以看到自己的文章了呢。

弄一个趁手的写博客工具

使用atom可以方便的边写md格式边预览,就选他了

安装atom,然后下载相关的一些好用的插件

https://github.com/atom-community/markdown-preview-plus

Atom自带的Markdown预览插件markdown-preview功能比较简单，markdown-preview-plus对其做了功能扩展和增强。

打开/关闭实时预览 Ctrl + Shift + M

勾选Preview position synchronization behaviour下的三个选项，实现预览同步滚动。

https://github.com/burodepeper/language-markdown

代码编辑增强插件

https://github.com/nmecad/markdown-img-paste

使用截图工具将图片复制到系统剪切板。

在Markdown新起一行输入文件名。

Ctrl + V 会自动把图片保存到Markdown文件相同目录下(因此要求Markdown文件应该先保存)，并命名为刚输入的文件名，同时在刚才输入文件名行处生成img标签。

https://github.com/susisu/atom-markdown-table-editor

表格编辑辅助，有了他写表格更方便

https://github.com/platformio/platformio-atom-ide-terminal

Ctrl + ` 打开/隐藏终端窗口

可以方便的输入hugo server来启动服务测试，也方便git发布

https://atom.io/packages/atom-timestamp

每篇的博客开头带有lastmod的时间，表示最后修改的时间，使用这个插件可以在你保存的时候把这个时间更新掉，方便的很。

为了匹配格式，需要修改配置

TimeStamp Preifx 设置 lastmod:[ \t]+

TimeStamp Suffix 设置 $
